# 换货处理流程（5.4）

> 前置：已读 `docs/INDEX.md`，工单类型确认为「换货」
> 核验重点：**退回商品规格编码是否与原订单一致**

---

## 完整流程

### Step 1：读鲸灵工单详情

```bash
node cli.js read-ticket <工单号>
```

记录：
- `subBizOrderDetailDTO[0].spuBarcode` — 货号
- `subBizOrderDetailDTO[0].attribute1` — attr1
- 售后说明 + 是否有售后图片
- 客户退货快递单号（buyerLogisticsNo）

### Step 2：查商品明细（必查，不靠记忆）

```bash
# 货号 → 规格商家编码
node cli.js product-match <货号> "<attr1>" <ERP店铺名>

# 规格商家编码 → 单品信息（subItemNum、名称）
node cli.js product-archive <规格商家编码>
```

记录**原订单规格商家编码**（用于 Step 4 对比）。

### Step 3：ERP 售后工单新版搜退回快递单号

```bash
node cli.js erp-aftersale <退货快递单号>
```

展开所有已收货行，读取：
- shortTitle（商品简称）
- 规格商家编码
- 数量
- 良品/次品（qtyBad）

### Step 4：验收判断

```
ERP 收货规格商家编码 = 原订单规格商家编码
  + 数量吻合
  + 全良品（qtyBad = 0）
→ ✅ 同意换货

ERP 收货规格商家编码 ≠ 原订单规格商家编码
→ 进入 5.4a（退回物品不符）
```

> ⚠️ **禁止用商品名字判断是否一致**，必须用规格商家编码对比。

### Step 4a：退回物品不符

执行以下步骤（不点任何同意/拒绝按钮）：

```bash
# 1. 添加内部备注（写 shortTitle，不写编码）
node cli.js add-note <工单号> "退回商品为[ERP shortTitle][数量]个，与订单商品[原商品shortTitle]不符，需拒绝换货+[后续动作]"

# 2. 创建 Mac 提醒（5分钟后触发）
node cli.js remind <工单号> <账号名> "⚠️ 换货退回商品不符：{退回商品} vs {换货商品} → {动作}"

# 3. 停止对该工单一切后续操作，等待人工处理
```

> 备注示例：`退回商品为KGOS保温壶1个，与订单商品灵芝金花黑茶不符，需拒绝换货+人工补发杯子`

### Step 5：同意换货

```bash
node cli.js approve <工单号>
```

## 决策快速参考

```
鲸灵读详情 → 货号 + attr1 + 退货快递单号
  ↓
商品对应表+档案V2 → 原订单规格商家编码（必查）
  ↓
ERP售后工单新版搜退货快递单号 → 展开已收货行
  ↓
对比规格商家编码：
  ├─ 一致 + 数量吻合 + 全良品 → 同意换货
  └─ 不一致 → 备注+提醒+停止，等待人工
```
